按**现代化 Web 产品的交互标准**设计了以下原型页面。为了让场景更具象，这里用**「医疗 IT 与实施」**主题作为演示数据。

---

### 页面一：翻译大厅与任务生成 (Translation Hub)

这是用户进入「翻译模块」的第一视觉中心。

```text
=============================================================================
[导航栏] 首页 | 卡包(Decks) | 复习 | 🌟 场景翻译(当前)          [用户头像]
=============================================================================

                   【 沉浸式场景翻译练习 】
        突破哑巴英语，将语言转化为你的硬技能。

  ┌─────────────────────────────────────────────────────────┐
  │  当前练习主题： [ 🏥 医疗 IT 与实施  ▼ ] (点击下拉切换)    │
  │                                                         │
  │  难度偏好：( ) 基础表达  (•) 进阶长句  ( ) 骨灰级难度   │
  │                                                         │
  │          [ 🚀 动态生成并开始挑战 ]                      │
  └─────────────────────────────────────────────────────────┘

  ⬇️ 历史任务档案 (Archive & Review)
  -----------------------------------------------------------
  | 状态  | 生成时间    | 主题        | 源文本摘要             | 得分 | 操作    |
  | 🔴复习| 2026-04-26 | 奢侈品鉴定  | The distinctive...   | 75  | [重译]  |
  | 🟢已达| 2026-04-25 | IT 全栈开发 | Implement a React... | 92  | [查看]  |
  -----------------------------------------------------------
```

**🗣️ 交互逻辑说明：**

1. **主题切换**：点击下拉菜单，展示四大分类和具体主题（支持搜索）。
2. **生成机制**：点击「动态生成并开始挑战」后，按钮变为 `[ ⏳ 生成场景中...]`，后台调用 AI 并将生成的句子瞬间**写入数据库**（记录生成时间、主题、难度）。
3. **页面流转**：数据库写入成功后，页面平滑过渡到“翻译实战”页面。

### 页面二：翻译实战台 (Active Translation Workspace)

采用极简的专注模式，减少视觉干扰。

```text
=============================================================================
← 返回大厅 | ⏱️ 计时: 01:23 | 🏷️ 主题: 医疗 IT 与实施
=============================================================================

  【场景源文本】
  ┌─────────────────────────────────────────────────────────┐
  │                                                         │
  │  "我们需要在下周前将新的 HL7 数据接口部署到医院的生产服     │
  │  务器上，并确保不影响现有的门诊工作流。"                  │
  │                                                         │
  │                                         [ 🔊 朗读 ]     │
  └─────────────────────────────────────────────────────────┘

        💡 卡壳了？点击 [获取核心词汇提示] (扣除2分)

  【你的翻译】
  ┌─────────────────────────────────────────────────────────┐
  │ We need to deploy the new HL7 data interface to the     │
  │ hospital's production server before next week, and      │
  │ make sure it doesn't affect the current workflow.       │
  │_ (光标闪烁)                                              │
  └─────────────────────────────────────────────────────────┘

                  [ 提交并获取 AI 诊断 ]
```

**🗣️ 交互逻辑说明：**

1. **防抖与自动保存**：用户在输入框打字时，前端静默缓存在本地，防止误触刷新导致输入丢失。
2. **求助机制**：点击“获取提示”，弹出一个小气泡显示：`deploy`, `production server`, `outpatient workflow`，但结算时会扣除少许分数，平衡挑战性。
3. **提交动作**：点击提交后，输入框锁定。下方区域使用 **Streaming（流式输出）** 展开，逐字显现 AI 的批改过程，消除等待焦虑。

### 页面三：智能批改与沉淀 (AI Diagnostic & Knowledge Sync)

这是整个模块的灵魂，也是用户获得“爽感”的时刻。

```text
=============================================================================
【 AI 诊断报告 】
-----------------------------------------------------------------------------
📊 综合评分： 82 / 100  (超越了 65% 的进阶练习者)
Radar Chart: [准确度: 80] | [流利度: 85] | [词汇量: 80]

📝 逐句诊断 (你的版本)：
We need to deploy the new HL7 data interface to the hospital's production
server before next week, and [🟢 make sure] it doesn't affect the [🔴 current]
workflow.
• 🟢 Good: "deploy...to production server" 用词极其精准，符合 IT 工程规范。
• 🔴 Bad: "current" 略显平淡。原文强调“门诊”，遗漏了关键信息 "outpatient"。

✨ 润色优化 (修改后的你)：
We need to deploy the new HL7 data interface to the hospital's production
server by next week, ensuring it doesn't disrupt the existing outpatient workflow.
                                                    [ ➕ 提取生词制卡 ]
-----------------------------------------------------------------------------
👑 地道表达进阶 (Native Alternatives)：
【更正式/商务】
"It is imperative that we roll out the new HL7 data interface to the
hospital's production servers by next week, with zero disruption to the
existing outpatient workflows."
                                                    [ ➕ 整句生成填空卡 ]
=============================================================================
  ↓ 知识入库 ↓
  选择归属卡包：[ 无分类 (默认) ▼ ]  (可选)   [ 一键加入 Ebbinghaus 记忆库 ]

              [ 🔄 针对此句再译一次 ]     [ 🚀 生成下一个任务 ]
```

**🗣️ 交互逻辑说明：**

1. **视觉高亮**：使用颜色区分用户的亮点和错误，直接在用户的原句上做批注，直观清晰。
2. **一键制卡 (关键交互)**：
   - 选词：用户用鼠标高亮 `disrupt` 或 `outpatient workflow`，直接弹出制卡小窗（Front/Back），默认填入。
   - 整句提取：点击按钮，系统将原生表达自动转换为 Cloze 格式，例如：`It is imperative that we {{c1::roll out}} the new HL7 data interface...`。
3. **卡包选择（数据隔离）**：下拉框允许用户选择将其放入已有的 Deck（如“面试准备”）。**由于底层卡片模型的 deckId 是可选的**，用户如果懒得分类，保持“无分类”即可直接入库，系统底层直接绑定该用户的 UserId。

### 页面四：历史任务回顾与复习触发 (Archive & Spaced Repetition)

这个页面与咱们系统的核心算法紧密结合。

```text
=============================================================================
【 历史翻译档案 】 (支持按主题、得分筛选)
=============================================================================
[ 检索: 关键词 ]  [ 主题: 全部 ▼ ]  [ 状态: 需要复习 ▼ ]

📦 任务卡片视图：
┌────────────────────────────────────────────────────────┐
│ 🔴 需复习 (根据 SM-2 算法调度)                           │
│ 主题：DevOps 与系统运维 | 难度：进阶 | 上次得分：65分  │
│ 源文本：为了确保高可用性，我们通过 GitHub Actions 配置了自动..│
│                                      [ 立即重译挑战 ]  │
└────────────────────────────────────────────────────────┘
┌────────────────────────────────────────────────────────┐
│ 🟢 已巩固                                              │
│ 主题：日常社交 | 难度：基础 | 历史最高分：95分           │
│ 源文本：很高兴能参加今天的会议，希望能从大家身上学到很多。      │
│                                      [ 查看历史对比 ]  │
└────────────────────────────────────────────────────────┘
```

**🗣️ 交互逻辑说明：**

1. **算法调度**：这里直接复用 Ebbinghaus 算法。上次得分低的任务，或者到达遗忘临界点的任务，会打上 `🔴 需复习` 标签。
2. **重译挑战**：点击重译，进入「页面二」，但不会生成新文本，而是调用已保存的 `sourceText`。提交后，系统会展示本次评分与上次评分的 **折线图对比**，直观展示进步。
